# App Audit – April 28, 2026

## Scope checked
- Static verification commands (`typecheck`, `lint`, `build`).
- Role/permission workflow review for access logic consistency.
- High-level module workflow review from router/module configuration.

## Confirmed logical bug fixed
### 1) `manager` role mismatch in type system vs feature logic
**Problem:**
- Multiple screens gate privileged actions with `profile.role === 'manager'`, but `UserRole` excluded `manager`.
- This mismatch can silently break role-based defaults and made manager checks "impossible" to type system, masking workflow issues.

**Fix applied:**
- Added `manager` to `UserRole`.
- Added default manager module access in `permissions.ts`.
- Updated settings page local role union to include `manager`.

## Additional issues discovered (not fully fixed in this patch)
1. **Type-check debt is very high** (hundreds of TS errors), concentrated around generic table typing and role checks.
2. **Lint debt is very high** (700+ errors), especially `any` usage and unused symbols.
3. **Build succeeds**, but large bundle warnings exist (several chunks >500kb), indicating performance optimization opportunity.

## Enhancement recommendations
1. **Stabilize RBAC source of truth**
   - Keep one canonical `UserRole` type and derive UI role unions from it.
   - Add tests for role-to-module mapping (`resolveAccessibleModules`).

2. **Progressive typing hardening**
   - Start with shared table components (`DataTable`) to reduce cascading `unknown`/`Record<string, unknown>` incompatibilities.
   - Replace `any` in Supabase response paths with generated DB types.

3. **Workflow-level regression tests**
   - Add simple smoke tests for key flows: Sales Order -> Delivery Challan -> Sales Invoice -> Payment posting.
   - Add permission matrix tests for Admin/Manager/Accounts/Sales/Warehouse.

4. **Bundle performance pass**
   - Introduce manual chunking for heavy routes (CRM, Dashboard, XLSX, html2canvas).
   - Lazy-load reporting and document-generation paths.
